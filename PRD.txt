# April Fools Love Sim (가칭) — 개발 기획 문서

## 1. 프로젝트 개요

본 프로젝트는 만우절을 기념하여 제작하는 텍스트 기반 연애 시뮬레이션 웹 애플리케이션이다.
사용자는 여러 남자 캐릭터(실제 친구 기반)를 선택하여 자유롭게 채팅을 진행하며, 대화 내용에 따라 호감도가 상승 또는 하락한다.
호감도가 100에 도달하면 해당 캐릭터 공략 성공(꼬심)으로 처리된다.

본 서비스는 재미와 공유를 목적으로 하며, 복잡한 게임 시스템 없이 간단한 구조로 빠르게 구현하는 것을 목표로 한다.

---

## 2. 기술 스택

* Frontend: Next.js
* Styling: Tailwind CSS
* AI: OpenAI GPT API
* State: React state + localStorage
* Deployment: (선택) Vercel

---

## 3. 핵심 기능

### 3.1 캐릭터 선택

* 메인 화면에서 캐릭터 목록 표시
* 각 캐릭터는 다음 정보를 포함:

  * 이름
  * 프로필 이미지
  * 한 줄 설명
  * 현재 호감도
  * 공략 완료 여부

---

### 3.2 채팅 시스템

* 사용자는 자유롭게 메시지를 입력할 수 있음
* GPT API가 캐릭터 설정에 맞는 응답 생성

AI 응답은 다음 요소를 포함:

* 행동 (1개)
* 대사 (1~2문장)

행동은 UI에서 다음과 같이 표시:

* `**행동 내용**`

---

### 3.3 호감도 시스템

호감도는 캐릭터별로 독립적으로 관리하며 localStorage에 저장한다.

#### 기본 규칙

* 채팅 1회 왕복마다 +5

#### 호감 키워드

* 유저 메시지에 포함 시 키워드당 +7
* 최대 2개까지 적용

#### 비호감 키워드

* 포함 시 키워드당 -8

#### 제한

* 최소: 0
* 최대: 100

---

### 3.4 엔딩 (꼬심 시스템)

* 호감도 100 도달 시:

  * 공략 성공 처리
  * 성공 안내 모달 표시
  * 캐릭터 상태 isCleared = true

---

## 4. 화면 구성

### 4.1 메인 화면

* 캐릭터 리스트 표시
* 각 캐릭터의 호감도 표시
* 캐릭터 클릭 시 채팅 화면으로 이동

---

### 4.2 채팅 화면

* 메시지 리스트
* 캐릭터 이름 및 호감도 표시
* 행동 + 대사 표시
* 입력창 및 전송 버튼

---

### 4.3 성공 모달

* 호감도 100 도달 시 표시
* 성공 메시지
* 캐릭터의 마지막 대사
* 메인으로 돌아가기 버튼

---

## 5. 데이터 구조

### 5.1 캐릭터 정의

```ts
type Character = {
  id: string;
  name: string;
  profileImage: string;
  description: string;
  personality: string;
  speechStyle: string[];
  funnyTrait: string;
  likeKeywords: string[];
  dislikeKeywords: string[];
};
```

### 5.2 상태 저장

```ts
type CharacterState = {
  affection: number;
  messages: ChatMessage[];
  isCleared: boolean;
};
```

```ts
type ChatMessage = {
  id: string;
  sender: "user" | "character";
  text: string;
  action?: string;
  createdAt: number;
};
```

localStorage key:

```
april_fools_love_sim_data
```

---

## 6. AI 응답 형식

GPT는 반드시 아래 JSON 형식으로 응답해야 한다:

```json
{
  "message": "대사",
  "action": "행동",
  "mood": "감정"
}
```

규칙:

* message: 1~2문장
* action: 반드시 1개
* 캐릭터 성격 유지
* 현실적인 말투 사용
* 과도한 표현 금지

---

## 7. AI 프롬프트 규칙

공통 규칙:

* 너는 남자 대학생 캐릭터다
* 주어진 성격과 말투를 유지한다
* 답장은 짧고 자연스럽게 한다
* 장난스럽지만 공격적이지 않게 말한다
* 행동 1개 + 대사를 JSON으로 반환한다

---

## 8. 호감도 계산 로직 (프론트 처리)

1. 기본 점수 +5
2. likeKeywords 포함 여부 체크 → 최대 2개까지 +7씩
3. dislikeKeywords 포함 여부 체크 → -8씩
4. 최종 점수 적용
5. 0~100 범위로 제한

---

## 9. UX 원칙

* 빠르게 반응 (대기 시간 최소화)
* 메시지는 짧고 템포 있게
* 현실적인 대화 느낌 유지
* 친구끼리 공유 가능한 수준의 가벼운 유머
